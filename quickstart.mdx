## إعداد التطوير الخاص بك

تعرف على كيفية تحديث المستندات محليًا ونشرها للجمهور.

### التحرير والمعاينة

<AccordionGroup>
  <Accordion icon="github" title="استنساخ المستندات محليًا">
    خلال عملية التأهيل، قمنا بإنشاء مستودع على Github الخاص بك مع
    محتوى المستندات الخاصة بك. يمكنك العثور على هذا المستودع على
    [لوحة التحكم](https://dashboard.mintlify.com) الخاصة بنا. لاستنساخ المستودع
    محليًا، اتبع هذه
    [التعليمات](https://docs.github.com/en/repositories/creating-and-managing-repositories/cloning-a-repository)
    في الطرفية الخاصة بك.
  </Accordion>
  <Accordion icon="rectangle-terminal" title="معاينة التغييرات">
    تساعد المعاينة في التأكد من أن تغييراتك تظهر كما هو مقصود. لقد قمنا ببناء
    واجهة سطر الأوامر لعرض هذه التغييرات محليًا.
    1. قم بتثبيت [Mintlify CLI](https://www.npmjs.com/package/mintlify) لمعاينة
    تغييرات الوثائق محليًا باستخدام هذا الأمر:
    ```
    npm i -g mintlify
    ```
    2. قم بتشغيل الأمر التالي في المجلد الرئيسي للوثائق (حيث
    يوجد ملف `mint.json`):
    ```
    mintlify dev
    ```
  </Accordion>
</AccordionGroup>

### نشر تغييراتك

<AccordionGroup>

<Accordion icon="message-bot" title="تثبيت تطبيق Github الخاص بنا">
  يقوم تطبيق Github الخاص بنا تلقائيًا بنشر تغييراتك على موقع المستندات الخاص بك، لذلك
  لا تحتاج إلى إدارة عمليات النشر بنفسك. يمكنك العثور على رابط التثبيت على
  [لوحة التحكم](https://dashboard.mintlify.com) الخاصة بك. بمجرد تثبيت الروبوت
  بنجاح، يجب أن تكون هناك علامة صح بجانب رقم الالتزام
  الخاص بالمستودع.
</Accordion>
<Accordion icon="rocket" title="دفع تغييراتك">
  [قم بالالتزام ودفع تغييراتك إلى
  Git](https://docs.github.com/en/get-started/using-git/pushing-commits-to-a-remote-repository#about-git-push)
  لتحديث موقع المستندات الخاص بك. إذا قمت بالدفع ولم تر أن
  تطبيق Github قد نشر تغييراتك بنجاح، يمكنك أيضًا تحديث
  مستنداتك يدويًا من خلال [لوحة التحكم](https://dashboard.mintlify.com) الخاصة بنا.
</Accordion>

</AccordionGroup>
