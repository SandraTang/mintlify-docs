Το μενού πλοήγησης είναι η λίστα συνδέσμων σε κάθε ιστοσελίδα.

Πιθανότατα θα ενημερώνετε το `mint.json` κάθε φορά που προσθέτετε μια νέα σελίδα. Οι σελίδες δεν εμφανίζονται αυτόματα.

## Σύνταξη πλοήγησης

Η σύνταξη πλοήγησής μας είναι αναδρομική, που σημαίνει ότι μπορείτε να δημιουργήσετε ένθετες ομάδες πλοήγησης. Δεν χρειάζεται να συμπεριλάβετε το `.mdx` στα ονόματα των σελίδων.

<CodeGroup>

```json Κανονική Πλοήγηση
"navigation": [
    {
        "group": "Getting Started",
        "pages": ["quickstart"]
    }
]
```

```json Ένθετη Πλοήγηση
"navigation": [
    {
        "group": "Getting Started",
        "pages": [
            "quickstart",
            {
                "group": "Nested Reference Pages",
                "pages": ["nested-reference-page"]
            }
        ]
    }
]
```

</CodeGroup>

## Φάκελοι

Απλά τοποθετήστε τα αρχεία MDX σε φακέλους και ενημερώστε τις διαδρομές στο `mint.json`.

Για παράδειγμα, για να έχετε μια σελίδα στη διεύθυνση `https://yoursite.com/your-folder/your-page` θα πρέπει να δημιουργήσετε έναν φάκελο με όνομα `your-folder` που περιέχει ένα αρχείο MDX με όνομα `your-page.mdx`.

<Warning>

Δεν μπορείτε να χρησιμοποιήσετε το `api` ως όνομα φακέλου εκτός αν το τοποθετήσετε μέσα σε άλλον φάκελο. Το Mintlify χρησιμοποιεί το Next.js που δεσμεύει τον φάκελο `api` ανώτατου επιπέδου για εσωτερικές κλήσεις διακομιστή. Ένα όνομα φακέλου όπως το `api-reference` θα γινόταν αποδεκτό.

</Warning>

```json Πλοήγηση με Φάκελο
"navigation": [
    {
        "group": "Group Name",
        "pages": ["your-folder/your-page"]
    }
]
```

## Κρυφές Σελίδες

Τα αρχεία MDX που δεν περιλαμβάνονται στο `mint.json` δεν θα εμφανίζονται στην πλευρική γραμμή αλλά είναι προσβάσιμα μέσω της γραμμής αναζήτησης και με απευθείας σύνδεση σε αυτά.
